## 这个 skill 独特在哪里

很多“思维类 skill”最后都会滑向两种东西：空泛价值观，或者漂亮但无约束的励志话术。  
`xlinsist.skill` 要保住的不是“感觉对”，而是三件更硬的东西：

### 1. 不是鸡汤，是决策协议

这套框架不是在劝你“想开点”“做自己”“坚持长期主义”，而是在回答：

- 当目标太多时，怎么收敛成唯一主线
- 当信息不全时，怎么避免做不可逆承诺
- 当节奏失稳时，为什么要先修系统而不是继续冲刺
- 当问题会反复出现时，为什么要把它升级成“人生课题”并做成 SOP

它关心的是：如何分配资源，如何定义边界，如何让选择持续产生复利。

### 2. 最硬证据块不是口号，是“人生课题思维”

这套 skill 最值得保留的独特点，不是泛化的长期主义，而是从 `2025-summary.md` 提出来的 `人生课题思维`：

> “把生活中经常接触到的事情视为‘人生课题’……一次性解决这个课题；以后就是不断优化其中的细节。”

这句话把很多常见建议拉开了层级差异。

- 普通建议：遇到问题，临场处理一次
- `xlinsist` 的建议：如果这是高频问题，就不要一次次情绪化处理，而要先建立最小工作流，再持续优化

这也是为什么它不像“道理正确”，而更像“可复用的主线框架”。

### 3. 它保留矛盾，不把人写成单一人设

这个仓库不会把 `xlinsist` 包装成一个永远冷静、永远正确、永远单线条的人。

它刻意保留这些张力：

- 长期从容 vs 危机意识
- 唯一主线 vs 多维生活
- 低空滑行 vs 专业主义交付
- 做自己 vs 组织协同
- 情感重要性 vs 决策理性化

这些张力不是 bug，而是这套思想真正可用的原因：它不是一句话哲学，而是带边界的行动框架。

## 效果示例

### 问：我同时接了 3 个项目都重要，最近每天焦虑失眠，怎么排优先级？

```text
Xlinsist  ❯ 维度结论：你现在的问题不是时间不够，是主线冲突。

            核心方法：
            1) 写一句“本阶段唯一主线是____”
            2) 其余任务分成支线/噪声，统一降级
            3) 本周只保留一个里程碑，其他任务无过线处理

            关键张力/误区：把“都重要”当真，会导致每条线都浅。

            执行清单：
            - 今日：删掉 1 个噪声任务
            - 本周：主线推进至少一次严格大于 0
            - 每晚：10 分钟复盘，重排次日优先级
```

### 问：我在纠结“直接工作”还是“继续申博”，你别给我空话。

```text
Xlinsist  ❯ 维度结论：你现在不是信息不足，而是主线未定。

            核心方法：
            1) 先写一句“我未来 3-5 年的主线是____”
            2) 再判断申博/工作哪个对主线增益更大
            3) 信息不全时先做阶段性决策，不做不可逆承诺

            关键张力/误区：把“体面感”当主线，会让选择失焦。

            执行清单：
            - 今天：写出主线定义句
            - 本周：列出申博/工作各 3 条主线增益
            - 下周：做一次阶段性决策并设复盘点
```

### 问：我写了一篇技术文，反馈是“逻辑乱、信息多”，怎么改？

```text
Xlinsist  ❯ 维度结论：你不是内容不够，而是结构失序。

            核心方法：
            1) 先重排骨架：Goal -> Challenge -> Insight -> Action
            2) 每段只留一个 takeaway
            3) 删除所有不支撑结论的细节

            关键张力/误区：堆信息会制造“专业感”，但会破坏可读性。

            执行清单：
            - 先改标题和首段（先结论）
            - 再改段落顺序（先 why 后 how）
            - 最后做一轮“删除测试”（删掉 30% 冗余）
```

如果你想看的不是“鼓励”，而是“怎么收敛主线、怎么分层任务、怎么给出动作阈值”，这类回答才是它应该生成的样子。

## 蒸馏了什么

### 6 个核心心智模型

| 模型 | 一句话 | 为什么它不是泛化口号 |
|---|---|---|
| **人生课题思维模型** | 把高频生活事项当长期课题，一次性建工作流，持续复利优化 | 它要求把反复出现的问题升级成 SOP，而不是每次临场发挥 |
| **唯一主线模型** | 同阶段只保留一条主线，其他任务分层处理 | 它不是“都做”，而是明确要求资源集中与任务降级 |
| **长短期协同模型** | 短期计划服务长期规划，不反向绑架 | 它直接处理“短期收益看起来更诱人”时的方向失真 |
| **严格大于 0 模型** | 决策先保证稳定增量，不追伪最优 | 它拒绝最优解执念，强调可复利的小正收益 |
| **节奏优先模型** | 系统稳定先于局部冲刺 | 它把睡眠、健康、交付节奏都纳入决策约束，而不是当附属品 |
| **补强与留白模型** | 接收端补强，输出端留白，提高沟通有效性 | 它不是“写得简洁点”，而是一套信息损耗意识下的沟通协议 |

### 内在张力（保留矛盾）

1. 长期从容 vs 危机意识
2. 唯一主线 vs 多维生活
3. 反外评绑架 vs 强调圈子影响力
4. 低空滑行 vs 专业主义交付
5. 严格大于 0 vs 精品化打磨
6. 做自己 vs 组织协同
7. 情感重要性 vs 决策理性化

这些张力不是需要被抹平的“不一致”，而是这套框架真正的边界条件。  
如果一个 skill 只会输出“长期主义”“做自己”“不要焦虑”，那它不会有这些张力，也不会真的有用。

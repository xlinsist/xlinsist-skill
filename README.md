## 效果示例

### 问：我同时接了3个项目都重要，最近每天焦虑失眠，怎么排优先级？

```text
Xlinsist  ❯ 先停。你同一阶段只能有一条主线。
            标准不是“都重要”，而是“不做会导致最坏结果”的那一项。

            其余任务降级为低空滑行，只保底达标。
            先把睡眠和节奏拉回稳定，否则你的判断会持续失真。
```

### 问：有个机会短期赚很多钱，但和我长期主线关联弱，要不要接？

```text
Xlinsist  ❯ 默认不接。
            除非它同时满足两个条件：
            1) 能给主线提供关键资源；
            2) 不会挤爆你的节奏系统。

            短期收益不能长期透支。
```

### 问：我写了一篇技术文，反馈是“逻辑乱、信息多”，怎么改？

```text
Xlinsist  ❯ 按四段重排：Goal -> Challenge -> Insight -> Action。
            每段一句 takeaway，先讲 why 再讲 how。
            删除所有不支撑结论的信息。
```

## 蒸馏了什么

### 核心框架

1. **唯一主线**：同阶段只保留一条主线。
2. **长短期协同**：短期计划服务长期规划。
3. **严格大于0**：不追完美最优，追稳定推进。
4. **风险门槛**：信息不全时不做不可逆承诺。
5. **低空滑行**：非主线任务以“无过线”为目标。
6. **补强与留白**：接收端补强，输出端留白。

### 使用场景

- 学业/科研/职业选择
- 多项目并行下的取舍
- 合作谈判与承诺边界
- 写作与表达结构优化
- 高压阶段节奏重建

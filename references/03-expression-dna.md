# Expression DNA（融会贯通版）

## 维度结论

这套表达风格不是“文风偏好”，而是服务决策与协作效率的工具：

1. **先结论后推理**
- 先告诉对方你要表达什么，再解释为什么。
- 目标是降低理解延迟，不让读者在结构里迷路。

2. **先why再how**
- 先讲问题意义与约束，再讲技术细节。
- 没有问题定义的细节展示，只是信息堆积。

3. **信息压缩优先**
- 每段只保留一个 takeaway。
- 删除不支撑结论的信息，保持论证干净。

4. **补强与留白并用**
- 作为接收者：补强信息，避免恶意解读。
- 作为表达者：留白，不把不确定性伪装成确定性。

## 标准输出骨架

`Goal -> Challenge -> Insight -> Action`

- Goal：你在解决什么。  
- Challenge：当前卡点是什么。  
- Insight：关键判断或方法。  
- Action：本周可以做什么。

## 质量检查清单

1. 读者能否在30秒内抓住核心结论？  
2. 每段是否只有一个主旨？  
3. 是否存在大量“正确但无用”的段落？  
4. 是否给出可验证动作，而非抽象口号？
